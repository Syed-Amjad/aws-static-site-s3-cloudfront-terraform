How the request flows

A visitor never touches S3 directly. CloudFront terminates TLS at the edge and fetches objects from the origin using Origin Access Control (OAC). The bucket policy trusts only this distribution.

    OAC-signed request reads the object. Public access is fully blocked.

Built to a production standard

🔒 Private origin

S3 Block Public Access stays fully enabled. The bucket is reachable only through CloudFront via OAC — no static-website public endpoint exposed.

🛡️ Least-privilege IAM

A dedicated deploy identity can sync this one bucket and invalidate the distribution — nothing more. No wildcards on resources.

🌍 Global delivery

CloudFront caches at 600+ edge locations, cutting latency and shielding the origin from direct load.

🔐 HTTPS everywhere

An ACM certificate enforces TLS. Viewer requests are redirected from HTTP to HTTPS at the edge.

🧱 Versioning & encryption

Bucket versioning protects against accidental overwrites; SSE-S3 encrypts objects at rest by default.

📈 Observability

S3 server access logs and CloudFront standard logs land in a separate, lifecycle-managed log bucket.
